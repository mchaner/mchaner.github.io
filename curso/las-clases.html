# Las clases

Hasta el momento hemos codificado mediante un paradigma estructurado, pero esto presenta sus desventajas ante algunos escenarios. Para esto nacieron otros paradigmas, pero en especifico el orientado a objetos. Agrupando elementos conocidos como objetos.

# Objeto

Un objeto tiene dos partes fundamentales:

  • Características
    • Tamaño
    • Color
  • Comportamiento
    • Avanzar
    • Retroceder

Para homologar esto a la programación podríamos decir que:

  • Las características son los atributos o campos
  • El comportamiento son los métodos

La idea principal es crear objetos para ser utilizados en distintas áreas de la necesidad del negocio

Clase vs Objeto

La clase es el plano a partir del cual se creara un objeto y este tendrá atributos y métodos.

# Declaración de una clase

Esto se puede realizar desde el mismo archivo de clase o desde un archivo distinto manteniendo el namespace

{

}

Para acceder a sus métodos o atributos solo necesitamos instanciar la clase (implementación)

namespace AplicacionConsola
{
    class Program
    {
        static void Main(string[] args)
        {
            ClaseEjemplo claseEjemplo = new ClaseEjemplo();
            claseEjemplo.nombre = "Diego";
            string respuesta = claseEjemplo.Nombre();
        }       
    }

    public class ClaseEjemplo
    {
        public string nombre;
        public string Nombre()
        {
            return "Luis";
        }
    }
}
