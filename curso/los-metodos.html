# Los Métodos

En la programación, un método es una subrutina cuyo código es definido en una clase y puede pertenecer tanto a una clase, como es el caso de los métodos de clase o estáticos, como a un objeto, como es el caso de los métodos de instancia.

Estructura de un método:

/* static => modificador
     * void => tipo o retorno
     * Metod => nombre del método
     * () => donde enviaremos los parámetros
     */
    static void Method(int num, string texto)
    {
        //instrucciones de código
    }

    //Método sin parámetros, ni tipo
    static void Method()
    {
        //instrucciones de código
    }

    //Método sin parámetros, pero con tipo
    static int Method()
    {
        int num = 0;
        return num;
    }

    //Método con parámetros, pero sin tipo
    static void Method(int num)
    {
        Console.Writeline(num);        
    }

    //Método con parámetros y con tipo
    static int Method(int num)
    {
        int response = num;
        return response;        
    }

# Contexto o ámbito

Un contexto es una secuencia ordenada de propiedades que definen un entorno para los objetos residentes dentro de ella.

# Pasar por referencia

La palabra clave ref deja que un argumento sea pasado por referencia, en lugar de por valor. El efecto que tiene esta palabra clave dentro de la implementación de un método consiste en mantener varias referencias sobre un único valor dentro y fuera del método que se invoca

Paso por valor

using System;

public class Program
{
    public static void Main()
    {
        int numMain = 10;
        Console.WriteLine(numMain);
        Modificador(numMain);
        Console.WriteLine(numMain);
        Console.ReadKey();
    }

    static void Modificador(int numMod)
    {
        numMod = 20;
    }
}

# Uso de "out"

using System;

public class Program
{
    public static void Main()
    {
        string nombre;
        int edad;

        Asignar(out nombre, out edad);

        Console.WriteLine("Me llamo {0} y tengo {1} años", nombre, edad);
        Console.ReadKey();
    }

    static void Asignar(out string nombreAsignar, out int edadAsignar)
    {
        nombreAsignar = "Chayanne";
        edadAsignar = 35;
    }
}

Variable con asignación implícita de tipo

Existe un tipo llamado var, el cual puede inferir que tipo que es dado el valor asignado.

var variable = "algo";
1

Para esto debemos tener algunas consideraciones:

  • Debemos Inicializar la variable.
  • No podemos declarar mas de una variable en linea.
  • Solo se pueden usar en ámbito de métodos, en instrucciones for o foreach.

Tipo de dato tupla

Esto nos sirve para agrupar mas de una variable dentro de una sola variable

(string,int,bool) variableAgrupada = ("texto", 50, true);
//imprimir por consola
Console.WriteLine(variableAgrupada); //("texto", 50, true)
Console.WriteLine(variableAgrupada.Item2); // 50

//(string valor,int,bool) variableAgrupada = ("texto", 50, true);
var variableAgrupada = (valor: "texto", 50, true);
Console.WriteLine(variableAgrupada.valor); // "texto"

# Los arreglos (array)

Un array (o arreglo) es una estructura de datos con elementos homogéneos, del mismo tipo, numérico o alfanumérico, reconocidos por un nombre en común. Para referirnos a cada elemento del array usaremos un índice (empezamos a contar por 0). Los arreglos igual los podemos asimilar a una matriz.

# Arreglos de una dimension

Los arreglos unidimensionales son los que cuentan con una sola fila de elementos

double[] notas;
string[] meses;
