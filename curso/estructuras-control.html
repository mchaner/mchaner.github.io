# C# como lenguaje de programación

Asi como para comunicarnos entre personas utilizamos algún lenguaje común para entendernos, también necesitamos un lenguaje en común para comunicarnos con la computadora y asi darle instrucciones para que nos logre entender. C# es un lenguaje de alto nivel orientado a objetos, lo que significa que su sintaxis es sencilla para cualquier persona, ademas es un lenguaje utilizado para crear aplicaciones de escritorio, web, mobile y videojuegos, sumado a esto tiene el respaldo de una gran compañía como microsoft.

# Variables

En el ámbito de los lenguajes de programación, una variable significa el espacio reservado en la memoria que tiene la capacidad de cambiar de contenido mientras se desarrolla la ejecución de algún software. Estas pueden almacenar un tipo de dato especificado y tener un nombre para identificarlas.

Consideraciones

  • Los identificadores deben tener un nombre con una letra o guion bajo.
  • Los identificadores pueden contener números, pero no empezar con ellos.
  • Los identificadores no pueden llevar acentos.
  • Los identificadores no pueden llevar símbolos extraños salvo el guion bajo.
  • Los identificadores no pueden llevar espacios.
  • Los identificadores no pueden ser palabras reservadas.
  • Los identificadores deben ser congruentes con su función.
  • C# es case sensitive.
  • Se utilizara Camel Case.

# Tipos y variables

Primero que todo debemos conocer un par de conceptos:

  • La pila o "stack": es una zona de memoria reservada para almacenar información de uso inmediato por parte del hilo de ejecución actual del programa. Por ejemplo, cuando se llama a una función se reserva un bloque en la parte superior de esta zona de memoria (de la pila) para almacenar los parámetros y demás variables de ámbito local. Cuando se llama a la siguiente función este espacio se “libera” (en el sentido de que ya no queda reservado) y puede ser utilizado por la nueva función. Es por esto que si hacemos demasiadas llamadas anidadas a funciones (en recursión, por ejemplo) podemos llegar a quedarnos sin espacio en la pila, obteniendo un “stack overflow”.

  • El montón o "heap": es una zona de memoria reservada para poder asignarla de manera dinámica. Al contrario que en la pila no existen “normas” para poder asignar o desasignar información en el montón, pudiendo almacenar y eliminar datos en cualquier momento, lo cual hace más complicada la gestión de la memoria en esta ubicación.

Ya con estos conceptos claros veremos:

  • Tipos por valor: son tipos sencillos que almacenan un dato concreto y que se almacenan en la pila. Por ejemplo, los tipos primitivos de .NET como int o bool, las estructuras o las enumeraciones. Se almacenan en la pila y se copian por completo cuando se asignan a una función. Por eso cuando se pasa un tipo primitivo a una función, aunque lo cambiemos dentro de ésta, el cambio no se ve reflejado fuera de la misma.

Lo que hacemos en el fragmento anterior es definir una clase persona muy sencilla y pasársela a una función que modifica su nombre. Cuando ejecutemos el código se verá por pantalla el nombre de la persona cambiado:

Es decir, los cambios son apreciables en el origen, fuera de la propia función.

¿A qué es debido esto? El motivo es que, como hemos dicho, los tipos por referencia se almacenan siempre en el montón, y lo que se pasa a la función como parámetro no es una copia del dato, como en el caso de los tipos por valor, sino una copia de la referencia al dato.

Para entenderlo de mejor manera veamos de manera gráfica; Vamos a ver en un gráfico la situación de la pila y el montón en el caso del primer código, donde solo manejábamos un tipo por valor:

Como podemos observar, al definir la variable i ésta se almacena en la pila directamente ya que es un tipo por valor. Al llamar a la función Suma2 pasándole i como argumento, lo que ocurre es que el valor de la variable i se copia a la pila y se asigna al parámetro n de la función (repasa el código anterior). De esta forma tenemos dos copias del dato en la pila, por eso al modificar n no se afecta para nada al valor original de i. En el montón, por cierto, no tenemos nada almacenado.

Consideremos ahora el código del segundo ejemplo y veamos qué se genera en la pila y en el montón en este caso:

Ahora la cosa cambia bastante. Al declarar una variable de tipo Persona se genera una instancia de la misma en el montón, ya que es un tipo por referencia. Al asignar los valores de las diferentes propiedades, esos datos se almacenan en el montón asociadas a la instancia de la clase Persona que acabamos de crear. Lo que realmente almacena la variable p es una referencia a los datos de esa persona que están en el montón.

TIP

Es decir, ahora en la pila no se almacenan los datos en sí, sino tan solo un “puntero” a los mismos. Es una forma indirecta de referirse a ellos, no como antes en los tipos por valor que el dato se almacenaba directamente en la pila.

Al llamar a la función CambiaNombre, en la variable local per que constituye el parámetro de la función, lo que se duplica ahora en la pila es la referencia al objeto, no el objeto en sí. Es decir, al hacer la llamada disponemos de dos variables que apuntan al mismo tiempo a los datos almacenados en el montón. Por eso cuando cambiamos una propiedad del objeto, al estar ambas referencias apuntando a los mismos datos, los cambios se ven desde los dos lados.

TIP

En realidad el comportamiento de la pila es idéntico en ambos casos, lo que cambia es la información que se almacena, que es el propio dato en el caso de los tipos por valor, y la referencia al dato en el caso de los tipos por referencia.

Este modo de funcionamiento es la diferencia fundamental que existe entre los tipos por valor y los tipos por referencia, y es muy importante tenerla clara pues tiene muchas implicaciones a la hora de escribir código.

# Operadores

Operadores aritméticos: Los operadores siguientes realizan operaciones aritméticas con operandos de tipos numéricos:

  • Unarios: ++(incremento), --(decremento), +(mas) y -(menos)
  • Binarios: * (multiplicación), / (división), % (resto), + (suma) y - (resta).

Estos operadores se admiten en todos los tipos numéricos enteros y de punto flotante. La jerarquía de operaciones es igual a la matemática.

using System;

namespace AplicacionConsola
{    
    class Program
    {
        static void Main(string[] args)
        {
            int numero = 5;
            //incremento
            Console.WriteLine(++numero);
            //decremento
            Console.WriteLine(--numero);
            //suma
            Console.WriteLine(20 + 15);
            //resta
            Console.WriteLine(20 - 8);
            //multiplicación
            Console.WriteLine(13 * 8);
            //división
            Console.WriteLine(6 / 4);
            //modulo
            Console.WriteLine(6 % 4);

            Console.ReadKey();
        }
    }
}

# La toma de decisiones IF - SWITCH

# IF

Tanto en la vida real como en la programación nos veremos enfrentados a tomar una decision para ver que camino debe seguir cierto evento, esto quiere decir que algo deberá suceder si se cumple cierta condición o también cuando no se cumpla.

Para resolver este tipo de decisiones necesitamos conocer los operadores de igualdad y de comparación los cuales se evaluaran y deberán entregar un resultado de condición boleana (true o false).

using System;

namespace AplicacionConsola
{    
    class Program
    {
        static void Main(string[] args)
        {
            int a = 4;
            int b = 6;

            // False
            Console.WriteLine(a == b);
            //True
            Console.WriteLine(a != b);

            int c = 7;
            int d = 7;

            //True
            Console.WriteLine(c == d);
            //False
            Console.WriteLine(c != d);

            Console.ReadKey();
        }
    }
}

# SWITCH

A diferencia del if (que evaluá una condicion), el switch compara una condicion con múltiples casos (expresión de coincidencia). Entendiendo que es similar a un if si tenemos mas de dos opciones condicionales se recomienda usar switch.

using System;

namespace AplicacionConsola
{
    class Program
    {
        static void Main(string[] args)
        {
            int num = 2;
            string numero;
            switch (num)
            {
                case 1:
                    numero = "uno";
                    break;
                case 2:
                    numero = "dos";
                    break;
                case 3:
                    numero = "tres";
                    break;
                case 4:
                    numero = "cuatro";
                    break;
                case 5:
                    numero = "cinco";
                    break;
                default:
                    numero = "error";
                    break;
            }

            Console.WriteLine(numero);

            Console.ReadKey();
        }

    }
}

Constante

Todo lo contrario a una variable (que su valor puede cambiar durante la ejecución del programa), la constante mantiene su valor inalterable, esto nos puede servir para definir ciertos valores que necesitemos que no cambien como por ejemplo pi (3.1416). Para definir pi como constante lo podemos realizar de la siguiente manera.

const decimal pi = 3.1416m;
